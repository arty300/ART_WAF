# ACADEM WAF

WAF (Web Application Firewall) с reverse proxy и интерактивной 3D-картой здания.
Все параметры настраиваются через веб-интерфейс — никаких конфиг-файлов вручную.

---

## Быстрый старт (Docker — рекомендуется)

### 1. Клонировать / скопировать проект

```bash
git clone <url> academ_waf
cd academ_waf
```

### 2. Запустить

```bash
docker compose up -d --build
```

После этого доступно:

| Адрес                   | Назначение                                      |
|-------------------------|-------------------------------------------------|
| http://localhost:9090   | Веб-интерфейс (карта, логи, настройки)          |
| http://localhost:8080   | WAF-прокси (сюда направляй трафик)              |

### 3. Остановить

```bash
docker compose down
```

Данные (БД) сохраняются в Docker volume `waf_data` — при повторном запуске всё на месте.

## Быстрый старт (без Docker)

### Требования

- Go 1.22+

### Сборка и запуск

```bash
cd academ_waf
go build -o waf ./cmd/waf
WAF_DB=./waf.db WAF_WEB=./web ./waf
```

## Настройка через веб-интерфейс

Открой **http://localhost:9090**.

### Вкладка «Настройки»

Здесь задаётся, куда WAF пересылает трафик.

| Поле            | Описание                                                            | Пример                    |
|-----------------|---------------------------------------------------------------------|---------------------------|
| Proxy Target    | Адрес твоего upstream-сервера                                       | `http://192.168.1.10:80`  |
| Proxy Listen    | Порт, на котором слушает WAF-прокси (требует перезапуска)           | `:8080`                   |
| Admin Listen    | Порт веб-интерфейса (требует перезапуска)                           | `:9090`                   |

После сохранения **Proxy Target** применяется мгновенно — следующий запрос уже идёт на новый адрес.
Смена портов применяется только после `docker compose restart`.

### Вкладка «Карта» — назначение IP по этажам и стенам

1. Открой вкладку **Карта**.
2. Наведи курсор на нужную стену здания (она подсветится).
3. Кликни — откроется боковая панель.
4. Введи IP-адрес и опциональную метку (например, «Кабинет 301»), нажми **Добавить IP**.
5. Можно добавить несколько IP на одно ребро.

Структура здания:

```
Этажи: 1–6 (снизу вверх)
Стены на каждом этаже: 6 штук
  Стена 0 — С-В (северо-восток)
  Стена 1 — В   (восток)
  Стена 2 — Ю-В (юго-восток)
  Стена 3 — Ю-З (юго-запад)
  Стена 4 — З   (запад)
  Стена 5 — С-З (северо-запад)
```

Когда через WAF проходит запрос с назначенного IP — соответствующая стена мигает:
- Зелёный — 2xx (успех)
- Жёлтый  — 3xx (редирект)
- Красный  — 4xx/5xx (ошибка)

### Вкладка «Логи»

Real-time таблица всех запросов. Колонки:
- Время, IP, местоположение (этаж / стена), метод, путь, статус, задержка.

## Как направить трафик через WAF

Вместо того чтобы обращаться напрямую к твоему серверу, клиенты обращаются к WAF:

```
Клиент  →  WAF :8080  →  Upstream (твой сервер)
```

**Пример**: если твой сервер слушает на `192.168.1.10:80`, то:
1. В настройках WAF укажи `Proxy Target = http://192.168.1.10:80`
2. Клиентам/браузерам давай адрес `http://<ip-машины-с-waf>:8080`

## REST API (для автоматизации)

| Метод  | Путь                  | Описание                                      |
|--------|-----------------------|-----------------------------------------------|
| GET    | `/api/logs?limit=N`   | Последние N записей из лога                   |
| GET    | `/api/ip-map`         | Все назначения IP → этаж/стена                |
| POST   | `/api/ip-map/assign`  | Добавить IP: `{floor, edge, ip, label}`       |
| DELETE | `/api/ip-map/assign`  | Удалить IP: `{floor, edge, ip}`               |
| GET    | `/api/config`         | Текущая конфигурация                          |
| PUT    | `/api/config`         | Обновить конфиг: `{proxy_target, ...}`        |
| GET    | `/api/stats`          | Счётчики запросов по IP за последний час      |
| WS     | `/ws/logs`            | WebSocket-стрим событий в реальном времени    |
